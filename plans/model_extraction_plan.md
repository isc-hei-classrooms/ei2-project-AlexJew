# Model Extraction Plan

## Context
The user requested to move the model training logic (Ridge regression and the simple baseline LightGBM) out of the `notebooks/energy_prediction.py` notebook and into separate Python scripts within the `scripts/` directory. 

The models should be generated by these scripts, saved to the `models/` directory, and then loaded by the notebook purely for evaluation and comparison.

Currently, the notebook computes features, splits the data, applies warmup clipping (Section 6.2), trains a Ridge model (Section 6.4), trains a LightGBM model (Section 6.5), runs Optuna tuning (Section 6.6), and then persists the data and models (Section 6.7).

## Challenges
- **Data Persistence:** The notebook currently persists the datasets and the `model_features.json` list *after* training the models in Section 6.7. To train the models in standalone scripts, the data persistence step needs to be moved *before* the model training steps so the scripts have data to load.
- **Model Loading:** The notebook currently uses the trained model objects directly in memory. It will need to be refactored to read the `.joblib` and `.txt` model files from the `models/` directory instead.
- **Kaggle Script:** We still have the pending Kaggle tuning script (`tune_lgbm_kaggle.py`) from the previous plan. This script replicates the feature engineering pipeline.

## Proposed Strategy

1. **Adjust Data Persistence in Notebook:**
   - Move the dataset persistence logic (`df_train_...parquet`, `df_test_...parquet`, and `model_features_...json`) to a new cell immediately after Section 6.2 (Warmup clipping).
   - Instead of appending timestamps to the exported files each time the notebook runs (which makes it hard for scripts to find the "latest" file easily without a glob search), we will update the persistence logic to save to fixed filenames for local dev/script use (e.g., `data/df_train_latest.parquet`, `models/model_features_latest.json`), and perhaps keep the timestamped ones as backups if requested.

2. **Create Training Scripts:**
   - Create `scripts/train_ridge.py`:
     - Load `data/df_train_latest.parquet` and `models/model_features_latest.json`.
     - Perform standardization.
     - Train `RidgeCV`.
     - Save the scaler and the model to `models/scaler_latest.joblib` and `models/ridge_latest.joblib`.
   - Create `scripts/train_lgbm_baseline.py`:
     - Load the same data and feature list.
     - Train the baseline LightGBM model.
     - Save the model to `models/lgb_default_latest.txt`.

3. **Refactor Notebook for Evaluation:**
   - Replace the training cells in Section 6.4 and 6.5 with cells that load `models/ridge_latest.joblib`, `models/scaler_latest.joblib`, and `models/lgb_default_latest.txt`.
   - If the files are missing, the notebook can print a warning instructing the user to run the scripts first.

4. **Address Section 6.6 (Optuna Tuning):**
   - The user previously wanted to move this to Kaggle (Commit 4 of the old plan). Since the feature data is now cleanly exported to `parquet` locally, the Kaggle script `tune_lgbm_kaggle.py` can either replicate the feature engineering (as originally planned) OR it can just expect the pre-computed `df_train_latest.parquet` and `model_features_latest.json` to be uploaded to Kaggle as a dataset. The latter is much cleaner. I will update the Kaggle script to just consume the pre-computed parquets.
